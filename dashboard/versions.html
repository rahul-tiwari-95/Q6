v1 Phase 1 — Single-Agent Baseline Dec 2025 · 6,000 eps
Thesis

A vanilla Double DQN with dueling heads and CNN state encoding can learn a viable evasion + collection policy against a scripted (non-learning) Hunter in a 25×25 grid world.

Architecture
  • CNNDuelingQNetwork: conv (32→64) → FC 256 → advantage + value heads
  • 6-channel binary state: Krishna pos, Hunter pos, pellets, walls, lives, steps
  • Single agent. Hunter is scripted greedy (deterministic pathfinding).
  • ε-greedy exploration: 1.0 → 0.05 over 6000 eps
Key Results
pellets/ep ~0.8
win rate ~12%
avg100 ~+15
Failure Mode

Against a scripted Hunter, Krishna over-fits to the deterministic pathfinding pattern. Performance collapses when Hunter is replaced with any learned policy. Not a general policy.

→ Need a co-evolving opponent. Next: self-play with Fictitious Self-Play (FSP) pool so Hunter is never predictable.
v2 / v3 Phase 2 Early — Self-Play Init + Bug Fixes May–Jun 2026 · ~3,000 eps each
Thesis

Introduce FSP (Fictitious Self-Play): maintain a pool of historical Hunter snapshots and train both agents simultaneously. Hypothesis: Krishna will generalize because it faces diverse Hunters, not just the current one.

Architecture Changes
  • SelfPlayEnv wrapping the gridworld — both agents step simultaneously
  • FSP pool: FIFO queue of 20 Hunter snapshots, snapshot every 100 eps
  • 70% joint training (latest Hunter) / 30% FSP (pool sample)
  • v3: fixed critical bugs — epsilon not decaying, position update race condition
Key Results
avg100 ~+4
pellets/ep ~0.2
self-play stable ✓
Failure Mode

Pool fills with strong Hunters too fast — the easy early Hunters get evicted before Krishna has learned to collect. Once pool=20 with all hard Hunters, Krishna's optimal strategy is pure evasion (survival > collection).

→ The pool architecture needs a permanent easy tier. Also: a single policy head can't learn both "evade" and "collect" simultaneously under adversarial pressure. Need dual heads.

Thesis

With bug-fixed self-play and a full FSP pool, Krishna should converge to a balanced evasion+collection policy. Hypothesis: more training time against diverse opponents resolves the collection paralysis.

Architecture

📊 Run details ▶ Replay
Failure Mode — Bimodal Strategy Collapse

Krishna learned evasion perfectly but collected almost zero pellets. Root cause: the FSP pool evicted all weak early Hunters before Krishna ever practiced collection. Once pool=20 hardened Hunters, risk of collecting (hunter nearby = death penalty ~−80) always outweighed pellet reward (+50). Single policy head cannot separate "evade" from "collect" — they interfere.

→ Need dual policy heads (evade head + collect head) with a learned gate. Need counterfactual experience injection to teach collection without real-world risk. Need permanent easy tier so Krishna always has safe practice time.
v5 Phase 3 — Gated Option Policies + Counterfactual HER Jun 2026 · 12,000 eps · 53.5h
Thesis

A dual-head gated architecture (evade head + collect head, context-conditioned gate) combined with counterfactual hindsight experience replay (CHER) will break the bimodal collapse. CHER injects "what if you had collected that pellet" transitions to teach collection without requiring the agent to risk it in practice. A hierarchical opponent pool with a permanent easy tier ensures collection is always practiced.

Architecture Changes
  • GatedOptionNetwork: CNN(6→32→64) → FC(256) → evade head + collect head + value head + gate MLP(context[3]→16→1→σ)
  • CounterfactualHER: post-episode scan; if hunter_dist > 6 and pellet_dist ≤ 4, inject +30 collect transition
  • HierarchicalOpponentPool: 5 permanent easy slots + 15 FIFO hard slots; p_easy=0.25
  • ContextReplayBuffer: stores (state, context, action, reward, next_state, next_context, done)
  • Context vector: [hunter_dist_norm, pellets_norm, lives_norm]
  • Warm-started from v4 best checkpoint (ep 5300)

What Worked
  • Gate never collapsed to pure evasion (v4 failure mode eliminated)
  • First 2-pellet episode (ep 10170, r_k=101.70) — dual-head proven
  • Late-phase avg100=8.46 mean / 15.69 max (vs v4's ~4–6)

Failure Mode — CHER Dependency / Reward Asymmetry
  • Against hardened pool=20 Hunter, collection still drops to ~3–5% per episode
  • CHER still firing 237k injections — agent never internalized autonomous collection
  • Gate oscillated 0.55→0.22→0.55 between ep 4000–5000 (instability)

→ The dual-head proof works but needs stronger gradient. The reward function itself is the bottleneck: dying costs too much relative to collecting. Need survival shaping (+0.1/step), proximity reward (continuous pellet gradient), gate regularization (prevent oscillation), and a 2-phase curriculum (collect-first, then harden).
v6 Phase 3 v2 — Reward Reshaping + Curriculum Planned · ~12,000 eps
Thesis

The v5 architecture is correct but the reward signal is broken. If we fix the reward asymmetry (dying is disproportionately expensive), add a continuous proximity gradient toward pellets, and run a 2-phase curriculum (collect first, then introduce strong Hunters), Krishna will learn collection as a primary behavior rather than an opportunistic side-effect.

Planned Changes
  • Survival bonus: +0.1 per step (max +100 at 1000 steps) — makes staying alive AND collecting both positive
  • Proximity reward: +0.05 × (1 − nearest_pellet_dist / grid_size) per step — continuous gradient toward pellets
  • Phase A curriculum: first 1000 eps against easy tier only (p_easy=1.0) — build collection habit before hardening
  • Gate regularization: L2 penalty on gate deviation from 0.5 during ep 0–2000 — prevents wild oscillation
  • CHER weighted injection: reward ∝ hunter_dist / safe_dist — teaches calibrated risk rather than only "safe" opportunities
  • ε floor raised to 0.08 — more exploration against strong Hunter
